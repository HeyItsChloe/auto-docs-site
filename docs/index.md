# auto-docs-site

A GitHub Actions automation that keeps a live docs site up to date on its own — regenerating a
codebase overview and a changelog either **on every merge to `main`** or **on a weekly schedule**,
whichever mode is configured, and deploying the result to GitHub Pages.

This site documents its own repository. Nothing here is hand-written after the first commit —
[`codebase-overview`](/codebase-overview) and [`changelog`](/changelog) are both regenerated by
the workflow in [`.github/workflows/docs-site.yml`](https://github.com/HeyItsChloe/auto-docs-site/blob/main/.github/workflows/docs-site.yml).

## How it works

1. A trigger fires — either a push to `main`, or the weekly `schedule`. Which one is "live" is
   controlled by a repo variable (`DOCS_TRIGGER_MODE`), so the same workflow file supports either
   mode without editing YAML.
2. A generation script runs in one of two engines, also configurable per run:
   - **`deterministic`** — walks the repo, builds a real file/folder tree and a lightweight
     import graph, no API key required.
   - **`llm`** — sends a representative slice of the codebase to the Claude API and asks for a
     structured architecture summary.
3. The changelog script diffs commits since the last recorded SHA and either lists them raw
   (`deterministic`) or groups them into Added/Changed/Fixed sections (`llm`).
4. VitePress builds the static site from the generated markdown, and it deploys to GitHub Pages.

See the [README](https://github.com/HeyItsChloe/auto-docs-site#readme) for setup — the repo
variables and optional `ANTHROPIC_API_KEY` secret that control all of the above.
